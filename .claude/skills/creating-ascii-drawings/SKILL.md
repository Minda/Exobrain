# Creating ASCII Drawings

## Purpose
Create visual ASCII diagrams to capture complex conceptual relationships, making abstract ideas tangible and memorable.

## When to Use
- User asks for a visual representation of concepts
- Complex relationships need clarification
- Creating a memorable summary of multi-faceted ideas
- Mapping tensions, agreements, or flows between concepts

## How to Create

### 1. Identify the Core Structure
- What's the main relationship? (comparison, flow, hierarchy, tension)
- What are the key entities or concepts?
- What connections need to be shown?

## Examples from Practice

### Epistemological Tensions
- Two groups separated by vertical line
- Shared agreements shown with horizontal connectors
- Divergences shown in parallel columns
- Integration point at bottom

### Conceptual Maps
- Central concept with radiating connections
- Hierarchical levels showing abstraction
- Arrows showing influences and dependencies

## Using Emojis Effectively

### When to Add Emojis
- To quickly signal the domain (ğŸ”¬ for scientific, ğŸ’­ for philosophical)
- To compress repeated concepts (instead of writing "risk" multiple times, use âš ï¸)
- To highlight key transitions or transformations (ğŸ”„)
- To mark important nodes or decision points (ğŸ¯)

### When NOT to Use Emojis
- If they make the diagram cluttered
- When ASCII characters are clearer
- If the emoji meaning isn't immediately obvious
- When precision matters more than brevity

### Balance
The goal is information density without sacrificing clarity. One well-placed emoji can replace several words, but too many create visual noise.

## Quality Checks
- Is the core relationship immediately visible?
- Can someone understand it without explanation?
- Does it add clarity beyond text description?
- Is it simple enough to remember?
- Do emojis enhance or distract from understanding?
